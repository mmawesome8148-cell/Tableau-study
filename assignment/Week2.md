## 10강: 차원과 측정값

> **🧞‍♀️ 차원과 측정값의 고유한 특성에 대해 설명해주세요.**

```
데이터를 연결하면 데이터 열을 필드로 만들고, 데이터 유형에 따라 필드를 차원 또는 측정값으로 할당함. 

'차원으로 분류되는 데이터'는 정성적인 값을 갖고 있는 데이터 
- 이름, 카테고리처럼 집계되거나 계산되지 않는 독립적인 필드
- 불연속형 필드: 개별적으로 구분되는 값

'측정값으로 분류되는 데이터'는 정량적인 수치값을 갖는 데이터
- 매출, 수익처럼 집계될 수 있는 값들
- 연속형 필드: 단절이 없는 무한한 범위의 값

그러나 차원=불연속형 필드, 측정값=연속형 필드 라고 생각하면 안됨. 
주문날짜의 경우 시각화 방법에 따라 불연속형, 연속형 둘다 가능
따라서 시각화 방법에 따라 적절한 필드를 사용할 줄 알아야 함. 
```

*Quiz: 차원은 불연속형이고, 측정값은 연속형이다 (o, x)*
```
답: x
```


## 11강: 시각화

```
(예1) 주문 날짜별로 매출 데이터 시각화
- 방법 1) 차원과 측정값을 시트로 갖고오면 자동으로 시각화 만들어줌
- 방법 2) 선반 활용, 열-주문날짜(차원)/ 행-매출 합계(측정값)
- 방법 3) 주문날짜 누른 채 + Ctrl키 + 매출필드 클릭 -> 시트에 추가
- 방법 4) 마크 창 활용, 주문날짜를 열 선반에, 매출필드를 마크창의
          텍스트에 드롭

+ 표현 방식으로 더 효과적인 시각화 방법을 선택할 수 있음. 

(예2) 년도별 지역별 매출 시각화
- 방법) 주문날짜 필드 옆에 지역 필드를 끌어오면 끝
```

*Quiz: 차원은 대부분 불연속형이며 표 형태로 시각화했을 때 머리글로 표시되고, 측정값은 대부분 연속형이므로 표 형태로 시각화 진행했을 때 패널로 추가된다.(o, x)*
```

## 12강: 막대그래프

```
1. 세로 막대그래프
   (예1) 카테고리별 매출 시각화 
   - 방법) 제품 카테고리 필드를 열 선반에, 매출합계를 행 선반에 드래그


2. 가로 막대그래프: 세로 막대그래프의 행렬을 바꾸면 됨. 
   (예1) 카테고리별 매출 시각화 
   - 방법) 제품 카테고리 필드를 행 선반에, 매출합계를 열 선반에 드래그

+ 행렬 바꾸기 버튼으로 가로> 세로 막대그래프로 바꿀 수 있음.

3. 막대그래프 정렬 방법
   - 상단의 정렬 버튼 클릭
   - 선반에 있는 제품 카테고리 마우스 오른쪽으로 클릭해서 정렬 기준 수정

4. 막대그래프 색상 변경
   (예) 제품 카테고리별로 다른 색상으로 시각화
        - 방법) 제품 카테고리 필드를 마크 창의 색상으로 드래그
        + 시각화 정렬에 따라 색상 범례의 정렬도 같이 수정 가능
          - 방법) 색상 범례 오른쪽 클릭 후, 정렬 기준 변경
    
    + 특정 색상을 사용하고 싶은 경우 
      - 방법) 마크창의 색상 클릭 > 색상 편집 > 팔레트 > 원하는 색상 선택 (*모양 할당: 그라데이션으로 색상 선택 가능)

5. 막대그래프 위에 레이블(숫자/카테고리) 표시 방법
   (예1) 막대 위해 매출 표시
        - 방법) 매출 필드를 마크 창의 레이블로 드래그
   (예2) 막대 위해 제품 카테고리 표시
         - 방법) 제품 카테고리 필드를 마크 창의 레이블에 드래그

    + 레이블을 막대그래프에 표시하고 싶으면
      - 방법) 마크창의 레이블 > 맞춤 > 세로 항목에서 가운데 정렬 선택

6. 막대그래프 두께 조정
   (예) 매출 순대로 두께 조정
        - 방법) 매출 필드를 마크 창의 크기로 드래그

```



## 13강: 누적막대그래프

```
누적 막대그래프의 쓰임: 차원별로 측정값으 비중을 보여주기 위한 목적으로 사용됨. 
- 축을 활용하여 구성비의 병화를 설명할 수 있음

(예1) 카테고리별 주문날짜 매출 누적 그래프 (=연도 누적 그래프)
      - 1단계) 제품 카테고리, 주문날짜를 열 선반으로, 매출 필드를 행 선반으로
      - 2단계) 열 선반의 주문날짜 필드 클릭해서 마크 창의 세부정보로 드래그
      
      + 년도를 색상으로 구분하려면
        - 방법) 마크창에 있는 주문일자 필드를 색상으로 드래그

      + 그래프에 년도, 매출 레이블을 추가하려면
        - 1단계) 선반에 있는 매출 필드를 Ctrl 키 누른 채로 마크 창의 레이블로 드래그, 마크창에 있는 년도 필드도 레이블로 드래그
        - 2단계) 마크창의 텍스트 > 자세히 버튼 > 년도 필드 아래에 매출 필드  & 가운데 정렬하기
    
    + 카테고리별 년도 매출 -> '백분율'로 표현하기
       - 1단계) 행 선반의 매출 필드를 마우스 오른쪽 클릭 > 퀵 테이블 계산 > 구성 비율
       - 2단계) 행 선반의 매출필드 오른쪽 클릭 > 다음을 사용하여 계산 > 테이블(아래로) 
       - 3단계) 다시 Ctrl키 누른 채로 행 선반에 있는 매출 필드를 마크창에 있는 매출  필드 위로 오버랩 시켜주면 레이블로 백분율이 보임. 

       + 테이블 계산에서 기준 필드를 변경할 수 있음
        - 방법) 행 선반의 매출필드 오른쪽 클릭 > 테이블 계산 편집 > 다음을 사용하여 계산 아래의 값 중 '특정 차원' 클릭 > 제품 카테고리 해제 -> 주문일자별로 백분율이 계산됨.

```

## 14강: 병렬막대그래프

```
병렬 막대그래프: 기본 막대그래프를 그룹화해서 나타내는 차트

(예) 2010년 - 2014년 까지의 월별 매출 병렬 그래프로

1단계) 주문일자를 열 선반으로, 매출필드를 행 선반으로 > 마크창에서 자동을 막대로 변경
2단계) 열 선반에 있는 주문일자 좌측에 있는 + 버튼을 두번 누르고 > Ctrl키 누른 채로 년 필드와 분기 필드를 밖으로 날리면 월 필드만 남게 됨. 

+ 병렬그래프를 그릴 때 필드를 연속형으로 설정해야 각 막대의 수평위치를 고정하여 그룹을 형성할 수 있음. -> 즉 주문일자를 연속형으로 설정해야 Tableau는 2010년의 1월과 2014년의 1월을 구분할 수 있음.

+ 불연속형 필드 > 연속형 필드로 바꾸는 방법
- 방법 1) 테이블 원본에서 변경, 화면 좌측에 있는 테이블에서 주문일자 필드를 오른쪽 클릭 > '연속형으로 변환' 클릭 
  -> 단점: 시각화마다 데이터 형태를 변경해야하는 번거로움이 있음, 데이터 유형 충돌 발생 가능

- 방법2) 선반에서 데이터 유형 변경, 열 선반에 있는 주문일자 필드 오른쪽 클릭 > 불연속형 속성은 중간라인 위, 연속형 속성은 중간라인 아래에 위치함 (연속형 부분에서 월 선택)

- 방법3) 테이블에서 Ctrl키 누른채 주문일자를 선반으로 갖고 올 때, 창이 뜨는데 이떄 데이터 유형을 선택할 수 있음 (파란색-불연속형, 초록색-연속형 -> 이 경우, 초록색 월 클릭)

  + 색상으로 월 구분
    - 방법) 선반에 있는 월 필드 Ctrl키 누른채로 마크 창의 색상으로 드래그 
```

## 15강: 누적병렬막대그래프

```
(예1) 지역별 수익과 매출 데이터 -> 상품 카테고리별로 누적해서 막대그래프 형태로 시각화
     - 1단계) 지역을 열 선반에, 수익과 매출을 행 선반에 드래그 > 표현 방식에서 누적 병렬그래프 클릭
     - 2단계) 마크창의 '측정값 이름' 날려주고 > 테이블의 상품 카테고리 필드를 마크창의 색상으로 드래그
     - 3단계) 주문일자를 열 선반에 추가 > 마크 창에서 자동을 막대로 변경

    + 막대 크기 조정
      - 방법) 마크창에서 크기 > 원하는 크기로 변경 가능

(예2) 이중 축을 활용한 누적형 병렬 그래프 - 2010년의 할인과 매출을 주문일자 월별로 시각화
                                       - 할인은 지역에 따라, 매출은 상품 카테고리에 따라 누적 막대 그래프
      - 1단계) 테이블의 주문일자 필드를 필터 선반에 드래그 > 년도 클릭 > 2010년만 클릭 후 확인 (필터 선반 안 보이면 빠른 메뉴에서 필터 선반 조건 클릭)
      - 2단계) 주문일자 필드 열 선반으로 > 필드 놓기에서 월 클릭, 측정값 이름도 열 선반으로 > 할인과 매출필드는 행 선반으로 드래그
      - 3단계) 이중축 만들기 (*이중축: 두가지 데이터를을 하나의 축을 사용해 표현하는 방식 ) 행 선반의 매출 필드 오른쪽 클릭 > 이중축 클릭 > 마크창은 막대로 변경
      - 4단계) 마크창에서 측정값 이름 날리고 > 할인 필드의 마크 창열어서 테이블의 지역 필드를 색상으로 드래그하고 > 매출필드 마크창에는 상품 카테고리 필드를 색상에 드래그

```

## 16강: 라인그래프

```
라인차트: 데이터의 시간별 추세를 확인하거나 데이터를 통해 미래값을 에측하려는 경우에 많이 사용되는 시각화 방법

(예1) 주문일자별 매출
     - 방법) 년도 필드를 열 선반에, 매출필드를 행 선반에

(예2) 월별 매출
      - 방법) 예1 수행 후, 열 선반의 년도필드 + 버튼 두번 클릭 > 열 필드만 남기기 > 연속형 월로 변경

(예3) 매출 vs. 수익 비교 그래프 
      -> 이처럼 두 그래프를 비교하기 위해선 이중축을 사용해야
      - 1단계) 테이블에서 수익필드를 행선반으로 옮긴 다음 > 마우스 오른쪽 클릭 > 이중축 클릭
      - 2단계) 양쪽 세로축을 일치시키기 위해, 둘 중 하나를 마우스 오른쪽 클릭 > '축 동기화' 선택

      + 값에 따른 다른 표현방식
        - 매출필드, 수익필드 모두 마크창의 색상으로 드래그 > 색상 편집 들어가서 커스터마이즈
```

```
텍스트만으로도 지도를 표현할 수 있음.
지리적 역할은 필드의 각 값을 위도와 경도값에 연결함.
지리적 역할에 값을 부여하면 위도와 경도가 자동으로 생성되어 원하는 맵을 그릴 수 있음.

1단계) 매출을 마크창의 색상으로 드래그하면 낮은 매출을 기록한 주는 연한 색으로 높은 매출을 기록한 주는 진한 색으로 표시됨. 
2단계) 행 선반의 위도를 Ctrl키 누른채 옆으로 드래그하면 지도가 복사됨. 
3단계) 두번째 위도 마크창에서 매출필드 날리고 수익필드를 색상으로 드래그 > 다시 수익필드를 크기로 드래그 > 마크를 원으로
4단계) 다시 행선반으로 가서 마우스 오른쪽 클릭 > 이중 축 클릭

+ 다양한 위치를 표시하는 방법 - 마크 계층 활용 (매출 -주단위, 수익-도시단위)
  1단계) 테이블의 주필드 더블 클릭 > 도시필드를 시트로 드래그하면 계층 추가 팝업이 뜨는데, 추가 아이콘 위에 도시필드를 드롭
  2단계) 주 마크창에 매출필드를 색상에 드래그, 도시 마크창에는 수익을 색상,크기로 드래그
